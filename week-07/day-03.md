# 第35天：初版测试和反馈收集

<ProgressCheck dayId="week-07-day-03" />

## 🎯 今日目标

让真实用户测试你的作品，收集反馈。学会从反馈中分辨哪些该改、哪些可以保持。

## 🛠 准备清单（2分钟）

- [ ] 准备好作品（能展示/运行的版本）
- [ ] 确定测试者（至少2人：家人、朋友、远程亲戚都可以）
- [ ] 准备纸笔记录反馈

## 📖 教学案例（30分钟）

<StepCard badge="Step 1" title="确定测试者并发布" duration="5分钟">

- AI绘本：把绘本导出成PDF或图片，发给测试者看
- AI游戏：让测试者在电脑前玩5分钟
- AI助手：把Coze分享链接发给测试者

</StepCard>

<StepCard badge="Step 2" title="三问反馈法" duration="15分钟">

**请每位测试者回答三个问题：**

```
━━━━━━━━━━━━━━━━━━━━
反馈者：___________

1. 你觉得这个作品整体怎么样？

2. 哪里做得最好？（说一个具体的点）

3. 如果只改一个地方让它更好，
   你会改什么？
━━━━━━━━━━━━━━━━━━━━
```

**至少收集3-5份反馈。**

</StepCard>

<StepCard badge="Step 3" title="分析反馈" duration="10分钟">

**把收集到的反馈分成三类，用不同颜色的便签纸：**

| 类别 | 反馈举例 | 怎么办 |
|------|---------|--------|
| 🟢 表扬 | "很有趣！""画得很可爱""创意很好" | 保持，并找到为什么好 |
| 🟡 技术问题 | "这张图不太好看""机器人回复太慢" | 直接改——重画/优化设置 |
| 🔴 设计问题 | "有点看不懂""好像太短了" | 思考怎么调整——加说明/加内容 |

**重点看🟡和🔴类**——这些是可以具体改进的地方。

</StepCard>

## ✏️ 孩子动手（10分钟）

### 写一份"改进计划"

```
【我的改进计划】

根据反馈，我要改进这3个地方：

1. 问题：________________
   怎么改：______________
   预计用时：____分钟

2. 问题：________________
   怎么改：______________
   预计用时：____分钟

3. 问题：________________
   怎么改：______________
   预计用时：____分钟
```

这份计划就是明天和后天的"施工图"。
