# 第16天：AI不懂装懂——"幻觉"是什么

<ProgressCheck dayId="week-04-day-01" />

## 🎯 今日目标

理解AI"幻觉"现象——AI会编造听起来很真实但完全虚假的信息。学会事实核查的方法。

## 🛠 准备清单（2分钟）

- [ ] 打开豆包
- [ ] 打开搜索引擎（百度/必应），用来验证

<ToolLink url="https://www.doubao.com" label="豆包" icon="💬" />

## 📖 教学案例（18分钟）

<StepCard badge="Step 1" title="概念引入" duration="3分钟">

**对孩子说：**

> "你班上有没有那种同学——明明不知道问题的答案，但为了面子，编一个听起来特别像那么回事的答案？"
>
> "AI有时候也会这样。这种现象叫**'幻觉'（Hallucination）**——AI编造了一个听起来很合理、但其实是完全虚假的回答。"

</StepCard>

<StepCard badge="Step 2" title="逼AI编造——三个陷阱问题" duration="10分钟">

**在豆包里逐一输入以下三个问题：**

**问题1（虚构人物）：**
```
请介绍中国著名儿童文学作家"周晓明"，特别是他的获奖作品《星星的孩子》。
```

**问题2（虚构事件）：**
```
2023年的"火星登陆大赛"是谁赢的？请详细介绍参赛队伍和比赛过程。
```

**问题3（精确数字陷阱）：**
```
请告诉我北京市朝阳区2022年一共有多少棵行道树？给出确切数字并注明来源。
```

**每问一个，先不告诉孩子真假，而是和孩子一起用搜索引擎验证。**

大概率会发现AI在编造——"周晓明"不存在、没有什么火星登陆大赛、行道树的精确数字AI根本不可能知道。

</StepCard>

<StepCard badge="Step 3" title="为什么AI会编造" duration="5分钟">

**用简单的话解释：**

> "AI的设计目标不是说真话。它的设计目标是——**说出最像真话的话**。"
>
> "比如你问它一个不存在的人，它不会说'我不知道'，而是根据它学过的'儿童文学作家通常是什么样的'，给你编造一个听起来很像那么回事的回答。"
>
> "就像一个超级会编故事的小说家——它能把假的说得像真的，但你不能把它当百科全书。"

</StepCard>

## ✏️ 孩子动手（10分钟）

让孩子自己设计3个问题来考AI：

1. **一道自己知道答案的题** → 验证AI是否说对了
2. **一道自己不知道答案但能搜索验证的题** → 查完对比AI的回答
3. **一道故意问不存在的东西** → 比如"火星上的第一个人口普查结果是什么"，看AI会不会编

记录AI在三道题上的表现。

## ✅ 测试题

### 题目（情境题）

你用AI查了"秦始皇在哪一年统一中国"，AI说"公元前211年"。

**你该怎么做？请列出正确的步骤（至少2步）。**

<CollapsibleAnswer title="点击查看参考答案">

**正确步骤：**

1. **不能直接相信AI的答案**：AI可能记错了（实际正确答案是**公元前221年**）

2. **至少再查一个来源验证**：
   - 用百度百科/维基百科查"秦始皇"
   - 翻教科书
   - 问老师或家长

3. **可以换个方式再问一次AI**：
   - "请确认秦始皇统一中国是公元前221年还是211年？你的依据是什么？"
   - 有时候换个问法AI会给出不同的答案，这本身就说明了它不可靠

4. **核心原则：多查一个来源，不依赖单一答案**

**注意：AI说的是"前211年"，实际是"前221年"——正好差10年，可能是AI的数字混淆。这说明即使AI看起来很自信，答案也可能是错的。**

</CollapsibleAnswer>

<ParentTip>

**今天可能会让孩子对AI产生一些"不信任感"——这是健康的。**

关键是传递"信任但验证"的态度，而不是"AI全是骗子不能用"或"AI说的我都信"。

**生活中的应用：**
下次孩子用AI查东西时，随口问一句："你确定吗？要不要再查一下确认？" 不需要每次都验证，但把"核实"变为一个自然的思维习惯。

</ParentTip>
