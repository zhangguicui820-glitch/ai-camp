# 第8天：给AI一个"模板"——用例子引导

<ProgressCheck dayId="week-02-day-03" />

## 🎯 今日目标

学会用"样例"引导AI按照你想要的格式输出。掌握"不知道怎么让AI做，就自己做一遍给它看"的黄金法则。

## 🛠 准备清单（2分钟）

- [ ] 打开豆包

<ToolLink url="https://www.doubao.com" label="豆包" icon="💬" />

## 📖 教学案例（13分钟）

<StepCard badge="Step 1" title="直接要求——没有模板" duration="3分钟">

**在豆包输入：**

```
写五个谜语
```

看回复。通常会出现：
- 格式不统一
- 有的谜语质量差
- 答案放在不同位置
- 有的太长有的太短

**问孩子："如果你想要每个谜语都像'身披绿衣裳，住在池塘边，饿了吃害虫，跳高是冠军——谜底：青蛙'这样工整，AI能做到吗？"**

</StepCard>

<StepCard badge="Step 2" title="给一个示例——让AI模仿" duration="8分钟">

**对孩子说：**

> "如果你想要AI按照你想要的格式来写，最好最省事的方法就是——你先写一个给它看。"

**这次输入：**

```
请按照以下格式，帮我写5个不同的谜语：

【参考示例】
谜面：身披绿衣裳，住在池塘边。饿了吃害虫，跳高是冠军。
谜底：（青蛙）

【要求】
1. 每个谜语都严格按这个格式写
2. 谜底分别是：猫、太阳、雨伞、铅笔、闹钟
3. 每个谜面4句话，每句话不超过10个字
```

**对比两次回答：**

- 第一次：格式混乱、质量参差不齐
- 第二次：格式统一、整齐

**问孩子："为什么给一个例子就完全不一样了？"**

**口诀：**

> **"不知道怎么让AI做，就自己做一遍给它看。"**

这个原则在以后学编程时同样适用。

</StepCard>

<StepCard badge="Step 3" title="扩展——示例的力量" duration="2分钟">

告诉孩子：这个方法不仅可以用在写谜语上，还能用在：
- 写诗（先写第一段）
- 写作文（先写开头）
- 翻译（先翻译一句做参考）
- 做表格（先做第一行）

**任何一个"你要特定格式"的场景，都可以用一个示例搞定。**

</StepCard>

## ✏️ 孩子动手（15分钟）

**任务：把自己写的东西变成AI的模板**

1. **选一个内容类型**：
   - 写几个笑话
   - 写几个成语的解释
   - 写几个人物的简介
   - 写几个动物的介绍

2. **自己先写一个完整的示例**（这是关键——必须自己先写）

3. **把示例发给AI，让它按同样格式再写5个**

4. **检查**：AI是否严格遵循了你的格式？如果不是，哪里跑偏了？

5. **截图保存**到"第2周"文件夹

## ✅ 测试题

### 题目（创作题）

请你：

1. 自己先写一个**"动物谜语"**作为示例（谜面+括号里的谜底）
2. 把这个示例发给AI，让AI按照你的格式写出**5个不同的动物谜语**
3. 从AI写的5个里挑出**你认为最好的3个**
4. 用这3个谜语去考家长，看家长能猜出几个！

把你写的示例和AI生成的谜语都截图保存。

<CollapsibleAnswer title="点击查看参考答案（家长检查清单）">

**检查三个点：**

1. ✅ 孩子是否自己先写了一个示例？（这个最关键）
2. ✅ AI生成的5个谜语格式是否一致？
3. ✅ 孩子有没有拿最好玩的谜语来考你？（互动环节）

**如果格式不太一致**：引导孩子看看是不是示例本身不够清楚？是不是需要补充更多格式要求？

</CollapsibleAnswer>

<ParentTip>

**"给例子"这个技巧看似简单，但在实际工作学习中极其重要。** 很多成年人到现在都还没掌握——他们花很多时间用文字解释格式，其实给一个例子3秒就能说清。

今天建议把孩子的示例和AI生成的谜语打印出来，或者手写在卡片上，变成一套"家庭谜语卡"。看到自己的创作变成实物，会很有成就感。

**如果孩子写的示例不够好**，没关系——重点是"先写再让它模仿"这个流程。示例好不好不重要，重要的是孩子体验到了"我的示例→AI模仿→检查结果"这个循环。

</ParentTip>
