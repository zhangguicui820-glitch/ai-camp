## 📖 教学案例（18分钟）

<StepCard badge="Step 1" title="故意写一个烂指令" duration="5分钟">

**在豆包里输入：**

```
写一个故事
```

只有4个字。看AI写了什么。

**问孩子：**
- "这是你想要的吗？"
- "你觉得好看吗？"
- "如果满分10分，你给几分？"

多数情况下AI会写一个"安全"但完全随机、毫无特色的故事。

</StepCard>

<StepCard badge="Step 2" title="分析问题——AI为什么做得不好？" duration="5分钟">

**引导孩子站在AI的角度想：**

当AI只收到"写一个故事"这4个字时，它脑子里有无数个问题：
- 什么类型的故事？童话？冒险？科幻？恐怖？
- 多长？50字还是5000字？
- 给谁看的？大人还是小孩？
- 主角是谁？动物还是人？
- 什么风格？搞笑还是感人？

**"AI在这种情况下只能'瞎猜'。它猜出来的故事，当然不太可能刚好是你想要的。"**

**总结口诀（写在便签纸上贴屏幕旁）：**

> **4个W法则：**
> **W**hat — 做什么？（写故事 / 画画 / 解释）
> **W**ho — 关于谁？给谁看？（主角 / 读者）
> **H**ow — 怎么做？多长？什么格式？
> **W**hat style — 什么风格？（搞笑 / 感人 / 童话 / 科幻）

</StepCard>

<StepCard badge="Step 3" title="改写——用4个W写一条好指令" duration="8分钟">

**和孩子一起，把"写一个故事"改写成好指令：**

```
请帮我写一个童话故事，要求如下：

1. 主角：一只胆小的恐龙叫小绿
2. 故事主线：讲小绿怎么一步一步变勇敢的
3. 长度：300字左右
4. 读者：给8-10岁的孩子看
5. 风格：温暖、有趣，有一个让人微笑的结尾
```

**把两次的结果放在一起对比。**

问孩子：
- "差别大吗？"
- "第二次的故事哪里比第一次好？"
- "如果满分10分，第二次给几分？"

**"你发现没有——AI还是那个AI，你的指令变了，结果就完全不一样了。"**

</StepCard>

## ✏️ 孩子动手（15分钟）

做**两组对比实验**，每组都像上面那样：先写烂指令，再用4个W改写，对比结果。

### 实验A：画画对比（用即梦）

| 版本 | 指令 |
|------|------|
| 烂指令 | "画一个人" |
| 好指令 | 加入4个W：什么人？什么样子？在哪？什么风格？例如"画一个戴红帽子的老奶奶，坐在木椅上织毛衣，阳光从窗户照进来，温暖写实风格" |

### 实验B：知识问答对比（用豆包）

| 版本 | 指令 |
|------|------|
| 烂指令 | "介绍一下大象" |
| 好指令 | 加入4个W："请用9岁孩子能听懂的话介绍大象，介绍3个有趣的知识点，介绍完给我出3道测试题" |

## ✅ 测试题

### 题目

你的好朋友小明对你说："我想让AI写一首关于夏天的诗，我觉得只要说'写一首关于夏天的诗'就够了。"

**你会给小明什么建议？至少说两条。**

把你给小明的建议写下来。

<CollapsibleAnswer title="点击查看参考答案">

可以从以下几个方面给建议（至少说两条）：

1. **告诉AI你想要什么风格**：轻快的？安静的？热闹的？
2. **告诉AI多长**：4行？8行？还是更长？
3. **给一个例子让AI模仿**：先写两句示范
4. **指定用哪些夏天的元素**：蝉鸣、冰西瓜、游泳、冰淇淋、雷阵雨……
5. **告诉AI给谁看的**：给小朋友的还是给大人的？

答出任意两条即可。

</CollapsibleAnswer>

<ParentTip>

**今天的核心是建立"输入决定输出"的意识。** 这是整个提示词工程的基础。

两个对比实验很重要，一定要做。孩子的学习方式是"看到差距"而不是"被讲道理"。当他看到"写一个故事"和4个W版本的天壤之别，道理自然就懂了。

**如果孩子写4个W指令时卡住了**，用以下问题逐一引导：
- What：你到底想让AI做什么？（先想清楚）
- Who：关于谁？主角是谁？
- How：多长？什么格式？有什么特殊要求？
- What style：想要什么感觉？

不要直接帮孩子写，让他自己一个个回答这四个问题。

</ParentTip>
