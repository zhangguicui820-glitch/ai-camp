# 第11天：教AI认东西——什么是"训练"

<ProgressCheck dayId="week-03-day-01" />

## 🎯 今日目标

亲手经历完整的数据标注→训练→测试流程，建立"训练"的直觉。核心认知：**数据越多、角度越丰富，AI学得越好。**

## 🛠 准备清单（3分钟）

- [ ] 打开 Teahable Machine
- [ ] 允许摄像头权限
- [ ] 准备两个不同的物品放摄像头前（比如水杯和遥控器、水果和玩具等）

## 📖 教学案例（28分钟）

<StepCard badge="Step 1" title="引入——AI就像婴儿" duration="3分钟">

**家长对孩子说：**

> "你有没有想过，你是怎么认识猫和狗的？是不是因为小时候，爸爸妈妈指着猫说'这是猫'，指着狗说'这是狗'，你看过很多次以后自己就会了？"
>
> "AI也是一样的。它一开始什么都不知道，需要人来'喂'给它很多例子，它才能自己找到规律。今天，我们就是AI的老师。"

</StepCard>

<StepCard badge="Step 2" title="数据标注和训练" duration="15分钟">

**操作步骤（和第3天类似，但训练任务升级）：**

1. 打开Teachable Machine → 选**"Image Project"**（图片项目）
2. 创建两个类别，比如：
   - Class 1：**杯子**
   - Class 2：**遥控器**
3. 把实物放在摄像头前，每个类别录制80-100张
   - **关键**：不同角度、不同距离、不同背景都要拍
   - 杯子的各种方向、正面侧面、远近距离都要有
4. 点击**"Train Model"**训练
5. 测试：拿实物放摄像头前，看AI能认对吗？

**讨论**：它学会了什么？——不是"理解"了什么是杯子，而是从几十张图片中总结出了杯子的共同视觉特征。

</StepCard>

<StepCard badge="Step 3" title="改进实验——数据量对比" duration="10分钟">

**做三组对比实验：**

| 实验组 | 每个类别的数据量 | 拍摄方式 | 预测结果 |
|--------|----------------|---------|---------|
| A组 | 只拍20张 | 同一角度 | 识别率低 |
| B组 | 拍100张 | 但角度单一 | 换个角度就认不出 |
| C组 | 拍100张 | 各种角度、距离、光线 | 识别率最高 |

**让孩子自己得出结论：**

> "数据越多、角度越丰富，AI学得越好。数据质量比数量还重要——100张全是同一个角度的，不如50张各种角度的。"

</StepCard>

## ✏️ 孩子动手（10分钟）

完成以上三组对比实验，记录每组的识别准确率。

记录格式：
- A组（20张/同角度）：对___次，错___次，准确率约___%
- B组（100张/同角度）：对___次，错___次，准确率约___%
- C组（100张/多角度）：对___次，错___次，准确率约___%

## ✅ 测试题

### 题目（应用题）

小丽训练了一个AI识别"苹果"和"橘子"。她在白色桌子上拍了100张照片。训练完在白色桌子上测试，准确率很高，很开心。

但当她把苹果和橘子拿到**厨房的木头桌子上**测试时，AI经常认错。

**你觉得问题出在哪里？怎么改进？**

<CollapsibleAnswer title="点击查看参考答案">

**问题出在哪里：**
AI把"白色桌子"也当成了一种特征来学习了。它学到的是"白色桌面上的圆东西=苹果"，而不是"苹果本身的样子"。

**改进方法：**
1. 在不同颜色的背景上都拍一些照片（白色桌子、木头桌子、蓝色桌子……）
2. 换不同光线环境下拍
3. 拿在手里拍、放在盘子里拍——变换场景
4. 这样AI才能学到"苹果本身的样子"，而不是"白色桌子上的苹果"

**总结：训练环境要多样化，测试才能适应各种环境。**

</CollapsibleAnswer>

<ParentTip>

今天和第3天的操作类似，但理解层次更深了——第3天是"体验训练"，今天是"理解训练的本质"。

**如果孩子问："AI是怎么从照片里学到规律的？"**

可以这样回答："大脑里有很多数学计算在自动运转。它先把每张照片变成一个数字矩阵（每个像素是一个数字），然后在这些数字中找规律——比如苹果通常是什么颜色范围、什么形状范围。这就是为什么照片越多越好，越多它越能找到稳定的规律。"

孩子不需要完全理解这段话，但"找规律"这三个字会种下一颗种子。

</ParentTip>
